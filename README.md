# SauceDemo Test Automation Framework

A robust and scalable Selenium-based test automation framework designed for the [SauceDemo](https://www.saucedemo.com/) e-commerce platform. This project follows the **Page Object Model (POM)** design pattern with **Page Factory** for better maintainability and code reusability.

## 🚀 Tech Stack

- **Language:** Java
- **Automation Tool:** Selenium WebDriver (v4.41.0)
- **Test Runner:** TestNG (v7.12.0)
- **Build Tool:** Maven
- **Driver Management:** WebDriverManager (v6.3.3)
- **Reporting:** ExtentReports (v5.1.2)
- **Pattern:** Page Object Model (POM) with Page Factory

## 📁 Project Structure

```text
SauceDemo/
├── src/main/java/
│   ├── base/               # Base class for Page Objects
│   ├── pages/              # Page Object classes (LoginPage, InventoryPage, etc.)
│   └── utils/              # Utility classes (DriverFactory, ExtentManager)
├── src/test/java/
│   ├── base/               # Base class for Test classes (Setup/Teardown)
│   └── tests/              # Test scripts and TestNG Listeners
├── reports/                # HTML reports generated by ExtentReports
├── testng.xml              # TestNG configuration file
└── pom.xml                 # Maven configuration and dependencies
```

## ✨ Key Features

- **Page Object Model (POM):** Clean separation between test logic and UI elements.
- **Centralized Driver Factory:** Optimized Chrome driver initialization with disabled password popups and maximized window settings.
- **Robust Base Classes:** `BasePage` and `BaseTest` handle common initializations and lifecycles.
- **Automated Workflows:** Includes comprehensive tests for Login, Add to Cart, and Checkout processes.
- **Dynamic Reporting:** Real-time test status reporting with ExtentReports, integrated via TestNG `ITestListener`.
- **Wait Mechanism:** Implements implicit waits for handling asynchronous elements.

## 🛠️ Prerequisites

- **Java Development Kit (JDK):** 11 or higher
- **Maven:** Installed and configured in system PATH
- **Browser:** Google Chrome

## 🏃 How to Run the Tests

### 1. Clone the Repository
```bash
git clone <repository-url>
cd SauceDemo
```

### 2. Run via Maven
To execute all tests defined in the `testng.xml`:
```bash
mvn test
```

### 3. Run via IDE
- Import the project as a **Maven Project** in Eclipse or IntelliJ IDEA.
- Right-click on `testng.xml` and select **Run As > TestNG Suite**.

## 📊 Test Reporting

After test execution, a detailed HTML report is generated in the `reports/` directory:
- **File:** `reports/ExtentReport.html`

The report provides a visual representation of test results, including pass/fail status and exception logs for failed tests.

## 🧪 Included Test Scenarios

- **LoginTest:** Verifies successful login with valid credentials and error handling for invalid ones.
- **AddToCartTest:** Validates adding products to the shopping cart and navigating to the cart page.
- **CheckoutTest:** End-to-end flow from product selection to successful order placement.
